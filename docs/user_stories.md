# 사용자 스토리 명세서 (User Stories)

> 물 섭취 습관 관리 앱 – 기록이 아닌 해석 중심

---

## 📌 개요

이 문서는 물 섭취 습관 관리 앱의 주요 기능을 **사용자 스토리(User Story)** 형식으로 정리한 요구사항 명세서입니다.

### 작성 원칙
- **As a (역할)**, **I want (기능)**, **So that (이유)** 형식 사용
- 로그인 기능은 제외 (MVP 범위 외)
- 물 마신 기록과 AI 기반 리포트 기능 중심

---

## 🎯 Epic 1: 물 섭취 기록 관리

### US-001: 간편한 물 섭취 기록
**As a** 바쁜 직장인  
**I want** ml 숫자를 입력하지 않고 버튼으로 간단하게 물 섭취를 기록하고 싶다  
**So that** 앱을 켜서 기록하는 것이 부담스럽지 않고 지속할 수 있다

**인수 조건 (Acceptance Criteria)**
- [ ] 3개의 선택 버튼 제공: `마셨음` / `조금 마셨음` / `거의 안 마셨음`
- [ ] ml 숫자 입력 필드는 존재하지 않음
- [ ] 하루에 여러 번 기록 가능
- [ ] 기록 시간이 자동으로 저장됨

---

### US-002: 기록 누락에 대한 부담 없음
**As a** 수험생  
**I want** 물을 마시지 못한 날이나 기록을 빼먹은 날에 대해 알림이나 경고를 받지 않고 싶다  
**So that** 실패했다는 죄책감 없이 앱을 계속 사용할 수 있다

**인수 조건**
- [ ] 하루 0회 기록도 허용됨
- [ ] 연속 기록(streak) 기능 없음
- [ ] 기록하지 않은 날에 대한 알림/경고 메시지 없음
- [ ] 누락일은 "데이터 없음"으로 조용히 처리

---

### US-003: 물 섭취 기록 조회
**As a** 사용자  
**I want** 내가 언제 얼마나 물을 마셨는지 날짜별로 조회하고 싶다  
**So that** 나의 물 섭취 패턴을 한눈에 파악할 수 있다

**인수 조건**
- [ ] 일별 물 섭취 기록 목록 조회 가능
- [ ] 기록 시간과 섭취 정도(마셨음/조금/거의 안 마셨음) 표시
- [ ] 최소 7일~30일 치 기록 조회 가능
- [ ] 기록이 없는 날은 빈 상태로 표시

---

### US-004: 주간/월간 물 섭취 요약 보기
**As a** 사용자  
**I want** 주간 또는 월간 단위로 나의 물 섭취 패턴을 시각적으로 보고 싶다  
**So that** 전체적인 흐름과 변화를 쉽게 이해할 수 있다

**인수 조건**
- [ ] 주간/월간 필터 선택 가능
- [ ] 날짜별 물 섭취 빈도를 시각적으로 표시 (차트, 캘린더 등)
- [ ] 기록이 없는 날도 시각적으로 구분됨
- [ ] 데이터가 없을 경우에도 안내 메시지 표시

---

## 🤖 Epic 2: AI 기반 해석 리포트

### US-005: AI 물 섭취 패턴 분석 리포트 생성
**As a** 컨디션 개선에 관심 있는 사용자  
**I want** AI가 나의 물 섭취 데이터를 분석해서 의미 있는 패턴과 변화를 설명해주기를 원한다  
**So that** 단순 숫자가 아닌 "내 몸과 생활에 어떤 변화가 있었는지"를 이해할 수 있다

**인수 조건**
- [ ] 최소 3일 이상의 데이터가 있을 때 리포트 생성 가능
- [ ] Gemini API를 활용한 자연어 해석 리포트 제공
- [ ] 리포트는 물 섭취 패턴, 요일/시간대 분석 포함
- [ ] 평가·훈계·실패 전제 언어는 사용하지 않음
- [ ] "관찰 → 해석 → 가벼운 제안" 순서로 구성

**출력 예시**
- "완벽하진 않았지만, 지난주보다 오후 물 섭취 빈도가 늘었어요."
- "평일 오전에 물 마신 기록이 많았고, 주말엔 오후에 집중되는 패턴이 보여요."

---

### US-006: 주간 AI 리포트 자동 생성
**As a** 사용자  
**I want** 매주 자동으로 AI 리포트가 생성되기를 원한다  
**So that** 별도로 요청하지 않아도 주기적으로 나의 변화를 확인할 수 있다

**인수 조건**
- [ ] 주 1회 자동으로 리포트 생성
- [ ] 생성된 리포트는 앱 내에서 조회 가능
- [ ] 리포트 생성 알림 제공 (옵션)
- [ ] 데이터가 부족할 경우, 부족함을 긍정적으로 설명

---

### US-007: 사용자 요청 시 즉시 리포트 생성
**As a** 사용자  
**I want** 내가 원할 때 언제든지 AI 리포트를 요청할 수 있다  
**So that** 궁금할 때 바로 나의 물 섭취 패턴을 확인할 수 있다

**인수 조건**
- [ ] "리포트 생성하기" 버튼 제공
- [ ] 최소 데이터 조건 충족 시 즉시 생성
- [ ] 데이터 부족 시 안내 메시지 표시
- [ ] 생성 중 로딩 상태 표시

---

### US-008: 과거 AI 리포트 조회
**As a** 사용자  
**I want** 과거에 생성된 AI 리포트를 다시 볼 수 있다  
**So that** 시간에 따른 나의 변화를 비교하고 돌아볼 수 있다

**인수 조건**
- [ ] 생성된 리포트 목록 조회 가능
- [ ] 리포트 생성일 순으로 정렬
- [ ] 각 리포트 클릭 시 전체 내용 확인 가능
- [ ] 최소 최근 10개 리포트 보관

---

## 💬 Epic 3: 컨디션 메모 (선택 입력)

### US-009: 컨디션 메모 기록
**As a** 사용자  
**I want** 물 섭취와 함께 오늘의 컨디션이나 느낀 점을 간단히 기록하고 싶다  
**So that** AI 리포트가 더 의미 있는 해석을 제공할 수 있다

**인수 조건**
- [ ] 하루 최대 1회 컨디션 질문 표시 (강제 아님)
- [ ] 선택 버튼 형태: `피로함` / `붓기` / `개운함` / `보통` 등
- [ ] 입력하지 않아도 앱 사용에 제약 없음
- [ ] 입력된 컨디션은 AI 리포트 분석에 활용

**질문 예시**
- "오늘 컨디션은 어땠어?"
- "붓기나 피로를 느꼈어?"

---

### US-010: 컨디션 메모 조회
**As a** 사용자  
**I want** 내가 기록한 컨디션 메모를 물 섭취 기록과 함께 보고 싶다  
**So that** 물 섭취와 컨디션 간의 관계를 스스로도 이해할 수 있다

**인수 조건**
- [ ] 물 섭취 기록 조회 시 컨디션 메모도 함께 표시
- [ ] 컨디션 메모가 없는 날은 표시하지 않음
- [ ] 캘린더나 목록 형태로 시각적 확인 가능

---

## 🔔 Epic 4: 맥락 기반 알림 (선택 기능)

### US-011: 알림 설정 (Opt-in)
**As a** 사용자  
**I want** 원할 때만 알림을 받고, 기본적으로는 알림이 꺼져 있기를 원한다  
**So that** 방해받지 않으면서도 필요할 때는 리마인더를 받을 수 있다

**인수 조건**
- [ ] 알림 기본값: OFF
- [ ] 설정 화면에서 알림 ON/OFF 가능
- [ ] 알림 시간대 지정 가능 (예: 오후 3~5시)
- [ ] 알림 톤: 권유형/허용형 (강요하지 않음)

**알림 메시지 예시**
- "지금 한 모금 정도면 충분해요"
- "오후 물 한 잔 어때요?"

---

### US-012: 맥락에 맞는 알림
**As a** 사용자  
**I want** 알림이 나의 하루 패턴을 고려해서 적절한 시점에 오기를 원한다  
**So that** 알림이 방해가 아닌 도움이 된다고 느낄 수 있다

**인수 조건**
- [ ] 사용자가 설정한 시간대 내에만 알림 발송
- [ ] 하루 최대 알림 횟수 제한 (예: 1~2회)
- [ ] 이미 물을 마신 직후에는 알림 발송 안 함

---

## 🎨 Epic 5: 사용자 경험 (UX)

### US-013: 최소한의 초기 설정
**As a** 새로운 사용자  
**I want** 앱 설치 후 복잡한 설정 없이 바로 사용을 시작하고 싶다  
**So that** 앱 사용의 허들이 낮고 즉시 가치를 느낄 수 있다

**인수 조건**
- [ ] 필수 입력 항목 최소화
- [ ] 온보딩 화면은 최대 2~3개 이내
- [ ] 바로 메인 화면(물 섭취 기록)으로 진입 가능
- [ ] 설정은 나중에 변경 가능

---

### US-014: 데이터가 적을 때도 가치 제공
**As a** 처음 사용하는 사용자  
**I want** 기록이 적어도 앱에서 의미 있는 피드백을 받고 싶다  
**So that** 데이터가 쌓일 때까지 기다리면서도 동기부여를 잃지 않을 수 있다

**인수 조건**
- [ ] 3일 미만 데이터에서도 간단한 격려 메시지 제공
- [ ] "데이터 부족"을 부정적으로 표현하지 않음
- [ ] "시작했다"는 것 자체를 긍정적으로 해석

**메시지 예시**
- "이제 막 시작이에요. 며칠 더 기록하면 더 재미있는 패턴을 찾을 수 있을 거예요."

---

## 📊 Epic 6: 데이터 관리

### US-015: 물 섭취 기록 수정/삭제
**As a** 사용자  
**I want** 잘못 기록한 물 섭취 내역을 수정하거나 삭제할 수 있다  
**So that** 실수로 잘못 입력해도 바로 정정할 수 있다

**인수 조건**
- [ ] 기록 목록에서 개별 항목 선택 가능
- [ ] 수정 시 섭취 정도 재선택 가능
- [ ] 삭제 시 확인 메시지 표시
- [ ] 수정/삭제 후 즉시 화면 반영

---

### US-016: 데이터 내보내기
**As a** 사용자  
**I want** 나의 물 섭취 기록과 리포트를 다운로드하거나 공유할 수 있다  
**So that** 다른 곳에서도 활용하거나 보관할 수 있다

**인수 조건**
- [ ] CSV 또는 JSON 형식으로 내보내기 가능
- [ ] 리포트는 텍스트 또는 PDF로 공유 가능
- [ ] 내보낸 데이터에 날짜, 시간, 섭취 정도 포함

## 📝 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 성능
- 앱 실행 후 메인 화면 로딩 시간 3초 이내
- AI 리포트 생성 시간 10초 이내
- 물 섭취 기록 저장은 1초 이내 완료

### NFR-002: 접근성
- 모바일 웹 및 데스크톱 웹 모두 지원
- 반응형 디자인 적용
- 다크 모드 지원 (선택 사항)

### NFR-003: 데이터 보안
- 사용자 데이터는 암호화하여 저장
- Supabase를 활용한 안전한 데이터 관리
- 외부 공유 시 개인정보 포함하지 않음

### NFR-004: 확장성
- 향후 목표 설정, 게임화 요소 추가 가능한 구조
- 다양한 AI 모델로 교체 가능한 아키텍처
- 다국어 지원 준비 (현재는 한국어)

- US-012: 맥락에 맞는 알림
- US-016: 데이터 내보내기

## 🚫 제외 사항

- **로그인 기능**: MVP에서는 제외 (추후 Supabase Auth 활용 가능)
- **소셜 공유/경쟁 기능**: 개인 중심 경험 우선
- **정확한 ml 단위 입력**: 무부담 경험 유지
- **게이미피케이션**: 실패 부담 증가 우려로 제외

---

## 📌 핵심 가치 재확인

> 이 앱은 **물을 더 많이 마시게 만드는 것이 목적이 아니라**,  
> **물을 마신 나의 상태를 이해시켜주는 것**이 목적입니다.

사용자는:
- 실패하지 않았다고 느끼고
- 변화를 언어로 이해하며
- 다시 앱을 켤 이유를 갖게 됩니다
